# Plans

Use this folder for project and initiative planning documents created from this template.

## When To Write A Plan

- If you start a new product or service from this template, create the first planning document here before the first feature implementation.
- For follow-up work, add more planning documents when the scope is large, the direction changes, or the team needs a clearer product decision record.

## Which Template To Use

- Start with `templates/project-brief.md` for the initial project brief.
- Use `templates/product-prd.md` when the work needs more detailed requirements, constraints, risks, or rollout planning.

## Naming Rule

- Create planning documents as `YYYY-MM-DD-<project-or-initiative>.md`.
- Keep the slug short and descriptive, such as `2026-03-12-b2b-admin-console.md`.

## Suggested Flow

1. Let `pnpm template:init` create the first dated brief from `templates/project-brief.md` during repository bootstrap.
2. Review and refine the minimum product context before implementation starts.
3. Expand into `templates/product-prd.md` only when the work needs deeper structure.
4. Update the planning document when major assumptions or scope decisions change.
